# 021 live pane repair

## 指示内容
- 現在の tmux ペイン構成が崩壊しているため、ライブ状態を修復する。

## 実施内容
- `tproj-workspace:dev` の全ペイン状態を確認。
  - `codex-pN` と `claude-pN` が横並び化し、幅 1 などの壊れた列を検出。
  - `claude-p7` の孤立列（対応 codex なし）を検出。
- ライブ修復を実施:
  1. `@column` 単位で片側しかない orphan 列を `kill-pane` で削除。
  2. 各列で `codex/claude` 幅が不一致または同一高さの場合、`join-pane -v` で縦ペアへ再構成。
  3. `~/bin/rebalance-workspace-columns tproj-workspace` を実行して横幅を再配分。
- 修復後の確認:
  - 6列すべてで `codex-pN`/`claude-pN` が縦構成に復旧。
  - 列幅は残差を含みつつほぼ均等（例: 62/62/62/62/62/66）。

## 課題、検討事項
- tmux の残差配分により完全同一ピクセル幅は保証されず、右端列が数文字広くなることがある。
- この修復はライブ実行済み。今後の再発防止は `rebalance` のタイミング改善（add/remove直後・agent不在時）で継続対応。
