# ask-codex ヘルプ

## 概要

Codex（OpenAI CLI）にtmux経由で質問を送信するスキル。
tprojセッション内で動作し、Claude CodeとCodex間のコミュニケーションを実現。

## 呼び出し方法

| 方法 | 例 | 説明 |
|------|-----|------|
| スキル名 | `/ask-codex` | 直近の作業から自動で質問を構築 |
| 引数付き | `/ask-codex この設計どう思う？` | 指定した内容を元に質問 |
| サブコマンド | `/ask-codex:status` | Codexペインの状態確認 |
| 自然言語 | 「Codexに聞いて」 | CCが自律的に質問を送信 |

## 使用条件

- tprojで起動したtmuxセッション内であること
- Codexペイン（dev.2）が存在すること
- Codexが起動していること

## レイアウト

```
┌─────────────────┬─────────────┐
│                 │   codex 30% │  <-- ここに質問が送信される
│     claude      ├─────────────┤
│                 │   yazi  70% │
└─────────────────┴─────────────┘
```

## 使用例

### 1. 設計の相談

```
/ask-codex このアーキテクチャについてどう思う？
```

### 2. セカンドオピニオン

```
/ask-codex 私はこの実装でいいと思うけど、見落としがないか確認してほしい
```

### 3. 自動生成（引数なし）

```
/ask-codex
```
直近の会話・作業から、Codexに相談すべき内容を自動で構築。

## 注意事項

- 質問はCodexが理解しやすいように再構築される
- Codexの回答完了を自動検知して結果を取得
- ログは `docs/log/codex/` に保存される
